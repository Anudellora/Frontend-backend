## Практическая работа №1 — Вёрстка карточек товаров на SASS

**Директория:** `practica1/`

### Описание
Создание страницы каталога товаров с использованием препроцессора **SASS (SCSS)** и методологии **БЭМ**.

### Что было сделано
- Разработана HTML-страница с каталогом из двух карточек товаров
- Написаны стили на SCSS с использованием:
  - **Переменных** (`$bg-page`, `$product-card-bg`, `$accent-color` и др.)
  - **Миксина** `product-card-base` для переиспользования стилей карточки
  - **Вложенной структуры** (nesting) в соответствии с БЭМ (`&__image`, `&__title`, `&__price`)
  - **Медиа-запросов** для адаптивности (breakpoint 768px)
- Реализована анимация при наведении на карточку (`transform`, `box-shadow`)
- Обеспечена доступность через `focus-within` и `loading="lazy"` для изображений

### Стэк технологий
- HTML5, SASS/SCSS, методология БЭМ

### Файлы
| Файл | Описание |
|------|----------|
| `index.html` | Главная страница с карточками товаров |
| `scss/style.scss` | Исходные стили на SCSS |
| `css/style.css` | Скомпилированный CSS |
| `img/` | Изображения товаров |

## Практическая работа №2 — CRUD API для управления товарами

**Директория:** `practica2/`

### Описание
Разработка серверного REST API и клиентской части для полного управления товарами (создание, чтение, обновление, удаление).

### Что было сделано

#### Серверная часть (`server/server.js`)
- Реализован Express.js сервер на порту 3000
- Созданы CRUD-эндпоинты:
  - `GET /api/products` — получение всех товаров
  - `GET /api/products/:id` — получение товара по ID
  - `POST /api/products` — создание нового товара
  - `PUT /api/products/:id` — обновление товара
  - `DELETE /api/products/:id` — удаление товара
- Реализована валидация входных данных (проверка обязательных полей)
- Обработка ошибок с информативными сообщениями
- In-memory хранение данных (массив `products`)

#### Клиентская часть (`index.html` + `js/app.js`)
- Динамическое отображение списка товаров через `fetch` API
- Форма добавления / редактирования товара (режим переключается автоматически)
- Кнопки «Изменить» и «Удалить» для каждого товара
- Защита от XSS через функцию `escapeHtml`
- SCSS-стили с переменными, миксинами и БЭМ-нотацией (лавандовая цветовая схема)
- Адаптивная вёрстка для мобильных устройств

### Стэк технологий
- **Backend:** Node.js, Express.js, CORS
- **Frontend:** HTML5, JavaScript (ES6+, Fetch API), SASS/SCSS, БЭМ
- **Инструменты:** Nodemon (dev-сервер)

### Запуск
```bash
cd practica2
npm install
npm run dev        # запуск сервера с nodemon
# Открыть http://localhost:3000/index.html
```

## Практическая работа №4 — Интернет-магазин ZVC (React + Express)

**Директория:** `practica4/`

### Описание
Разработка полноценного веб-приложения интернет-магазина **ZVC** (магазин игровых товаров) с разделением на клиентскую и серверную части.

### Что было сделано

#### Серверная часть (`server/index.js`)
- Express.js API с расширенной моделью данных продукта:
  - Поля: `id`, `name`, `category`, `description`, `price`, `stock`, `image`, `rating`
- CRUD-эндпоинты:
  - `GET /api/products` — все товаров
  - `GET /api/products/:id` — товар по ID
  - `POST /api/products` — добавление товара
  - `PATCH /api/products/:id` — частичное обновление товара
  - `DELETE /api/products/:id` — удаление товара
- Предзаполненный каталог из 10 товаров (плюшевые игрушки, манга, фигурки, лего и т.д.)
- Валидация обязательных полей

#### Клиентская часть (`client/`)
- **React SPA** с использованием Vite в качестве сборщика
- **React Router** для маршрутизации:
  - `/` — Главная страница (каталог товаров)
  - `/add` — Страница добавления нового товара
- Компоненты:
  - `Header` — навигация с активным состоянием ссылок
  - `ProductList` — список карточек товаров
  - `ProductCard` — карточка товара с изображением, категорией, описанием, рейтингом (звёзды), ценой и наличием на складе
  - `AddProductPage` — форма добавления товара с выбором категории из предустановленного списка
- Обработка состояний загрузки и ошибок
- CSS-стили для каждого компонента

### Стэк технологий
- **Backend:** Node.js, Express.js, CORS
- **Frontend:** React 18, React Router 6, Vite 5, CSS
- **Инструменты:** Nodemon, Concurrently

### Запуск
```bash
# Из корня проекта (frontend-second/)
npm run install:all    # установка всех зависимостей
npm run dev            # одновременный запуск клиента и сервера
```

## Практическая работа №7—8 — JWT Аутентификация

**Директория:** `practica9-10/` (базовый функционал)

### Описание
Реализация надежной системы аутентификации пользователей на основе **JWT (JSON Web Tokens)** с использованием пары токенов: **Access** (короткоживущий) и **Refresh** (долгоживущий).

### Что было сделано
- **Backend:**
  - Настроено хеширование паролей при регистрации пользователей с помощью `bcryptjs`.
  - Внедрена генерация пары JWT-токенов (`jsonwebtoken`) с поддержкой секретных ключей.
  - Реализованы эндпоинты для аутентификации:
    - `POST /api/auth/register` — регистрация нового аккаунта
    - `POST /api/auth/login` — вход в систему и получение токенов
    - `POST /api/auth/refresh` — обновление истекшего Access-токена с помощью Refresh-токена
    - `GET /api/auth/me` — получение данных текущего авторизованного пользователя
  - Разработан middleware `authMiddleware.js` для защиты приватных маршрутов.
- **Frontend:**
  - Создан компонент `AuthContext` для реактивного управления глобальным состоянием авторизации во всём приложении.
  - Настроено автоматическое сохранение и извлечение токенов из `localStorage`.
  - Установлены перехватчики (interceptors) в библиотеке Axios для автоматического подставления `Authorization`-заголовка и механизма прозрачного обновления токена при получении ошибки `401 Unauthorized`.

## Практическая работа №9—10 — Контроль доступа (RBAC) и Products API

**Директория:** `practica9-10/`

### Описание
Расширение системы интернет-магазина до полноценного управления товарами с глубоким разграничением прав доступа на основе ролей (**Role-Based Access Control**).

### Что было сделано
- **Система ролей (RBAC):** Внедрены 4 иерархических уровня доступа: 
  - `Гость` (только регистрация/вход)
  - `Пользователь` (просмотр товаров)
  - `Продавец` (добавление/изменение своих товаров)
  - `Администратор` (полное управление товарами и пользователями).
- **Backend (Products CRUD):** 
  - Реализованы защищенные эндпоинты для создания, чтения, изменения и удаления товаров.
  - Разработано кастомное middleware `roleMiddleware.js` для жесткого ограничения доступа на стороне сервера (например, эндпоинт `DELETE /api/products/:id` закрыт для всех, кроме роли `admin`).
- **Frontend:**
  - Интерактивная страница списка товаров с поиском и фильтрацией.
  - **Динамический UI:** Кнопки "Добавить товар", "Редактировать" и "Удалить" отображаются в интерфейсе только у тех пользователей, чья роль позволяет выполнить эти действия.
  - Разработан HOC-компонент `ProtectedRoute` для защиты и скрытия маршрутов на уровне `react-router-dom`. Поддерживает передачу массива `allowedRoles`.

## Практическая работа №11—12 — Панель администратора и API Tester

**Директория:** `practica9-10/`

### Описание
Создание комплексного интерфейса администрирования для полного управления всеми пользователями системы, а также внедрение утилиты интерактивного тестирования серверных запросов напрямую из UI.

### Что было сделано
- **Управление пользователями (User Management):**
  - Создан новый набор эндпоинтов `GET, PUT, DELETE /api/users` на сервере специально для администраторов.
  - Разработана страница **👥 Пользователи** для фронтенда (доступна только Администраторам).
  - Внедрена возможность мгновенной смены ролей пользователям через выпадающие списки прямо в панели.
  - Реализована функция "Бан" — блокировка/разблокировка аккаунтов пользователей, не позволяющая заблокированным лицам получить доступ к защищенным ресурсам в системе.
- **Интерактивный API Tester:**
  - Разработан продвинутый аналог Swagger внутри самого React-приложения для разработчиков.
  - Дает возможность тестировать абсолютно все эндпоинты сервера (Auth, Users, Products) без стороннего ПО.
  - Умная система: автоматическая подстановка актуального `Access` или `Refresh` токена в заголовки запросов, если пользователь залогинен.
  - Отображение cURL-команд для копирования в терминал, времени отклика (в мс), статус-кодов запроса и детальных ответов сервера (JSON).
- **UX/UI улучшения:**
  - Создана страница `UnauthorizedPage` (ошибка 403), если пользователь пытается зайти на маршрут без должной роли.
  - Статусные бейджи ролей в Navbar для визуального подтверждения текущих прав.
  - Добавлена возможность выбора роли при тестировочной регистрации.
